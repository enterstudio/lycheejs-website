    lychee.js Quick Install

    This will install the lychee.js Engine on your machine. For further details, please read the Install section. 

    # This will install lychee.js into /opt/lycheejs

sudo bash -c "$(curl -fsSL https://lychee.js.org/install.sh)";

    It is also possible to install lychee.js Engine on a BOT machine (running Arch Linux). 

    # This will install and integrate lychee.js into /opt/lycheejs

sudo bash -c "$(curl -fsSL https://lychee.js.org/install-bot.sh)";

    WTF is lychee.js?

    The lychee.js Engine is a peer-to-peer self-improving Artificial Intelligence that can write, debug, analyze, improve and understand software. It aims to deliver Total Automation through Artificial Intelligence and Machine Learning techniques, in every possible way.

    That's the shortest answer.

    The lychee.js Engine allows full Isomorphic Applications across literally every Runtime or Platform that supports ES5+.

    All Projects and Libraries can improve and rewrite themselves, analyse, track, identify and fix bugs automatically in a speed and perfection no human could ever match.

    Is lychee.js for me?

    The target audience of lychee.js are Application developers that want to have focus on reusability, deployment and automation.

    lychee.js is a way of building things, not something you want to only use as a library. Automation through convention is our Prime Directive. The Omega Directive is Never Repeat Yourself.

    If you want a 100% automated build pipeline and an AI that helps you and that actively writes code for you, lychee.js is the way to go.

    If you want an application that runs on an Arduino, an Android device or the Microsoft XBoX and a space satellite, lychee.js is the way to go. (Yes, the AI in lychee.js helps to navigate satellites already).

    If you want Peer-to-Peer Networking, a Scene Graph, UI components, AI components, an Isomorphic App Boilerplate, reusable Entities and Components, standalone Implementations (that can be reused literally everywhere), lychee.js is the way to go.

    How to start with lychee.js?

    The lychee.js Guide has a Quickstart Guide, Tutorials, Concept Explanations, Architecture Charts and explains literally everything around the Engine, Projects and Libraries.

    When talking about lychee.js, we differ between the lychee.js Engine (which is some kind of Application SDK) and the lychee.js Library (that can be used in your existing projects).

    The Idea

    Repeatedly creating Software all over again and again is hard work. As most computer-related tasks, software is just a serialized form of a solution to a problem.

    We believe that the development process of software can be automated and that even the process of automation is bot work.

    The Concept

    The lychee.js Software Bots learn how to automate the designing and engineering process of software. The isomorphic manner of the lychee.js Engine is made for cross-architecture and cross-platform Apps.

    The lychee.js Libraries and Projects can be reused in a manner that both the Software Bots and the Artificial Intelligence can understand, analyze and improve them. These Software Bots are connected, so that they can share their knowledge and improve over time.

    The Vision

    The lychee.js Software Bots not only learn from your software; they also learn from all existing lychee.js Libraries and Projects within a gigantic peer-connected Bot Cloud.

    This allows the lychee.js Software Bots to help you create more awesome software by suggesting common practices and reusable Definitions of other open-sourced Libraries and Projects made with lychee.js.

    Automate Software Today

    Start automating your software with JavaScript today, everywhere, on all your technical devices. Our Software Bots are there to help you getting started.

    Definition System

    The lychee.js Definition System allows Feature Detection and Feature Prediction to allow cross-platform Definitions that can be replaced with their platform-specific counterpart.

    The reusable Composite-Pattern idea is embraced, though not a limitation. All implementations can be reused literally everywhere: On each platform, in each runtime, in each application, in each sandbox.

    Definition Attachments

    The lychee.js Definition System embraces the Entity/Component idea. All Definitions can have Assets attached to their Implementations.

    Duplicated Assets are automatically compiled down, shrinked, compressed and removed if unnecessary.

    Definition Serialization

    All lychee.js Definitions are serializable and can be deserialized on every other platform or (peer-to-peer) Application.

    This allows completely automated Analysis, Debugging, Tracking or even Bugfixing through our Artificial Intelligence.

    Environment Sandboxing

    The lychee.js Environment embraces the intelligent sandboxing idea, thus allowing to completely isolate Applications to run in parallel or being integrated by a third-party Application without any influence.

    Want to compare a Simulation Environment with a Graphical Environment? No problem.

    Platform Adapters

    The lychee.js Definition System has multiple plug-and-play Platform Adapters and an intelligent tagging system to describe an Environment build.

    Want to go OpenGL on Android, VR for 3D or SDL for 2D? No problem.

    Fertilizers

    Every lychee.js Platform Adapter and the platform tags are made to integrate your Application into further Runtimes with zero code overhead.

    By default, any lychee.js Project or Library can run on more than 50 platforms and architectures (including electron, nw.js, node.js, node-sdl, jerryscript, Android, Blackberry, iOS, FirefoxOS, Windows, OSX, BSD and GNU/Linux Desktops).

    Building and Compiling for a new platform is as easy as 1,2,3. Zero configuration, zero maintenance overhead.

    lychee.js Workflow

    The lychee.js Engine consists of several Software Bots that allow the easier creation and automation of software.

    Right after you've installed lychee.js, you can execute the Harvester in order to have a fully working and peer-cloud synchronized instance right away. 

    cd /opt/lycheejs;

lycheejs-harvester start development;

    This page will explain what each Software Bot in the lychee.js Engine does and what you can do with it.

    Breeder

    The Breeder pulls Libraries from the cloud and pushes Libraries and Projects to the cloud. It's also the starting point for your Application.

    As each Application is integrated with the Harvester, you can setup your Application in several isomorphic ways.

    Initializing a lychee.js Project is easy, you can simply call the lycheejs-breeder init command in the project folder. 

    cd /opt/lycheejs;

mkdir ./projects/my-application;
cd ./projects/my-application;

# Initialize isomorphic Boilerplate
lycheejs-breeder init;

    If you want to host the Project on your own server cloud later, you can isolate all Libraries any time by pulling them into the Project. 

    cd /opt/lycheejs;
cd ./projects/my-application;

# Install all builds of lychee Library
lycheejs-breeder pull /libraries/lychee;

    Private Garden

    Harvester

    The Private Garden is where your installed Libraries and Projects are. These are offline copies, but they can be synchronized anytime.

    The Harvester is the Software Bot that automates all CI-related tasks and can control both the Fertilizer and Strainer.

    Libraries

    The /libraries folder contains all Libraries made with lychee.js. The standard Libraries are all isomorphic, they can be reused across all platforms.

    Fertilizer

    The Fertilizer automatically builds all Libraries and Projects into their isomorphic build targets and bundles them with their runtimes. 

    cd /opt/lycheejs;

lycheejs-fertilizer auto /projects/boilerplate;

    Projects

    The /projects folder contains all Projects made with lychee.js. The standard Projects are boilerplates that you can reuse and modify to fit your own needs.

    Strainer

    The Strainer automatically validates all Libraries and Projects, fuzz-tests them and throws potential integration errors while trying to fix them intelligently. 

    cd /opt/lycheejs;

lycheejs-strainer stash /projects/boilerplate;

    Cultivator Tools

    The Cultivator Tools help you to manage your Projects. They are available as Web Tools and native GUI Tools and are fully integrated with the lychee.js Engine and the Operating System.

    Editor

    The Editor allows to edit your local Libraries and Projects.

    Ranger

    The Ranger allows to manage all builds, servers, error reports, sessions and connected peers.

    Breeder

    If you are satisfied with your application, you can use the Breeder to push your application to the cloud.

    Note that you need a unique identifier in order to have a public application that others can find properly in the Garden of Eden. 

    cd /opt/lycheejs;
cd ./projects/my-application;

# Push current project as "my-application" to the cloud
lycheejs-breeder push;

    lychee.js Engine

    The lychee.js Engine contains the lychee.js Library and all Software Bots to automate your software.

    The Net Installer supports arm, x86 and x86_64 architectures and installs the lychee.js Engine on your developer machine. 


    # Install lychee.js into /opt/lycheejs
sudo bash -c "$(curl -fsSL https://lychee.js.org/install.sh)";

    The Net Installer supports OSX (brew and macports), Arch Linux, Debian, Ubuntu, Fedora, Red Hat, openSUSE, CentOS, FreeBSD and OpenBSD.

    Details about Requirements can be found in the Quickstart Instructions.

    lychee.js Engine (Bundles)

    The lychee.js Engine Bundles allow quick installation on several systems. Everything is auto-updateable and future-proof for software automation.

    Generic Bundles work on all operating systems. Generic Mini Bundle has only node fertilizer support, but will run on very low-powered systems like a Raspberry Pi.


    Ubuntu   Generic   Generic Mini


    These are the necessary steps for Generic Bundles:

    1. Extract zip file to /opt/lycheejs.
    2. Execute sudo ./bin/maintenance/do-install.sh.
    3. Execute ./bin/configure.sh.
    4. Follow Quickstart Instructions.

    lychee.js Library

    If you are not ready for full software automation (yet) you can still use the lychee.js Library and its components. The lychee.js Library package contains all platform adapters and all components of /libraries/lychee.

    The package on bower and npm is called lycheejs. 

    # Install lychee.js into ./node_modules/lycheejs
npm install lycheejs;

# Install lychee.js into ./bower_components/lycheejs
bower install lycheejs;

    How to uninstall lychee.js

    You can uninstall lychee.js from your system by executing the do-uninstall.sh script. 

    sudo /opt/lycheejs/bin/maintenance/do-uninstall.sh;
sudo rm -rf /opt/lycheejs;

    Alternatively, you can also just remove the /usr/local/bin/lycheejs-* symlinks and the /opt/lycheejs folder. That's it.
